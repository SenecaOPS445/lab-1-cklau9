1.  What is a git repository?

A Git repository is a storage location where a project's files and history are kept, allowing multiple people to collaborate on the same codebase.

2.  What kinds of data can be stored in a git repository?

A Git repository can store various types of data, including source code files, images, documents, configuration files, and any other type of file used in a project.

3.  What is a git branch?

A Git branch is a parallel version of a repository's codebase. It allows for the development of new features or bug fixes without affecting the main codebase (usually called the "master" or "main" branch).

4.  What is the name of the definitive branch in a git repository?

The definitive branch in a Git repository is typically called the "master" branch, although some projects now use "main" as the default branch name for improved inclusivity.

5.  What is a "pull request" related to a branch in a git repository?

A pull request is a request to merge changes from one branch into another. It's commonly used for code review and collaboration, allowing team members to discuss changes before merging them into the main codebase.
